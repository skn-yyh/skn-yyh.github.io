强引用（Strong Reference）

强引用是最常用的一个引用方式。如果如果程序存在一个强引用对象时，JVM 是不会回收该对象的，即便是当内存空间不足时，JVM 宁愿抛出 OOM 异常，也不会去回收这个对象去解决内存不足的问题。

软引用（SoftReference）

软引用在内存充足的情况下，不会被 JVM 回收，只有在内存不足的情况下才会被回收，避免OOM异常。

虚引用（PhantomReference）

虚引用是所有引用中最弱的一种引用类型。它不同于以上两种引用，无法通过 get() 方法获取被弱引用包装的对象。同样通过源码也可以看出方法的返回值是null。

那么虚引用到底有什么作用呢？虚引用主要用来跟踪对象被垃圾回收的情况。当 GC 触发以后，发现有虚引用后，会将虚引用包装的对象插入到 ReferenceQueue 队列中。此时，该对象并没有被垃圾回收器回收，而是处于准备回收的状态。

虚引用只有当垃圾回收器对其所指向的对象真正进行回收时，会将其加入到ReferenceQueue 对列中，这样就可以追究到对象回收情况。
